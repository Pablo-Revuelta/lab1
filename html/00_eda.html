Significant Earthquakes, 1965-2016. EDA¶

Se va a realizar un análisis EDA sobre los terremotos ocurridos de 1965 a 2016.

Se va a analizar las distintas variables y buscar relaciones entre ellas, con el objetivo de visualizar si las variables estan condicionadas entre sí.

Variables¶

En el dataframe nos encontramos con las siguientes variables:

  • Date: Fecha del evento. Tipo object.
  • Time: Hora del evento. Tipo object.
  • Latitude: Latitud del epicentro en grados decimales. Valores negativos para la latitud sur. Valores entre -90.0 y 90.0. Tipo float.
  • Longitude: Longitud del epicentro en grados decimales. Valores negativos para la longitud oeste. Valores entre -180.0 y 180.0. Tipo float.
  • Type: Tipo de terremoto. Tipo object.
  • Depth: Profundidad del evento en km. Valores típicos entre 0 y 1000. Tipo float.
  • Depth Error: Incertidumbre de la profundidad reportada en km. Valores típicos entre 0 y 100. Tipo float.
  • Depth Seismic Stations: Tipo float.
  • Magnitude: Magnitud del evento. Valores entre -1.0 y 10.0. Tipo float.
  • Magnitude Type: Método o algoritmo usado para calcular la magnitud. Tipo object.
  • Magnitude Error: Incertidumbre de la magnitud reportada.Se le da un valor NaN si no se reporta una incertidumbre. Valores entre 0 y 100. Tipo float.
  • Magnitude Seismic Stations: Tipo float.
  • Azimuthal Gap: Espacio entre estaciones. Cuanto menor es el número más exactitud en la posición horizontal del evento. Tipo float.
  • Horizontal Distance: Incertidumbre de la distancia horizontal hasta el epicentro del evento. Toma valor Nan si no se reporta nin´gun error en la estimación. Tipo float.
  • Horizontal Error: Incertidumbre de la localización del evento reportado. Se le da un valor NaN si no se reporta una incertidumbre. Tipo float.
  • Root Mean Square: Tiempo de viaje del evento, en segundos. Valores típicos entre 0.13 y 1.39. Tipo float.
  • ID: Identificador único para cada evento. Tipo object.
  • Source: Fuentes que reportaron el evento. Tipo object.
  • Location Source: Fuente que reportó la ubicación del evento. Tipo object.
  • Magnitude Source: Fuente que reportó originalmente la magnitud del evento. Tipo object.
  • Status: Si el evento ha sido revisado por un humano. Si el valor es Automatic es que se ha reportado de forma automatica y no ha sido revisado por un humano. Reviewed para eventos revisados por un humano. Tipo object.

Número de NaN por variable¶

Como se explica en el apartado de las variables, los valores NaN pueden estar condicionados a que no se ha reportado ninguna información debido a carencia de ella. Esto es porque no hay error en la estimacion de los eventos. Como se observa, las variables con mayor número de valores NaN son los que estiman algún tipo de error. Para estas variables se sustituiran los NaN por 0.

Para las variables Magnitude Seismic Stations y Depth Seismic Stations hay un numero considerable de errores. Se procederá a eliminar esta variable ya que no otorga información significante para analizar.

Para el resto de variables, sustituir los NaN por el valor 0 podría condicionar los análisis estadísticos y eliminar las columnas nos haría perder información interesante, por lo que los dejaremos como NaN.

Análisis correlaciones¶

Se observa muy poca correlación entre las variables. Las únicas variables con una mayor correlación son las variables que indican que se ha reportado un error en la estimación. Esto indica que cuando ocurre un error en la estimación de una de las variables con Depth, Magnitude u Horizontal, suele haber errores en la estimación de otras de las anteriores variables.
